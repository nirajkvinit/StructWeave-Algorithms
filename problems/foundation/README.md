# Foundation Problems

> Build computational thinking before tackling algorithmic patterns

**Foundation problems** are the entry point to StructWeave. They develop core programming skills and mathematical intuition that serve as prerequisites for the Easy, Medium, and Hard algorithm problems.

---

## What Makes Foundation Different?

| Aspect | Foundation | Easy (Algorithms) |
|--------|------------|-------------------|
| **Focus** | Computational thinking | Data structure patterns |
| **Skills** | Loops, modulo, sequences | Two pointers, hash maps |
| **Math** | Number theory, formulas | Minimal |
| **Data Structures** | Basic (arrays, sets) | Hash tables, stacks, trees |
| **Time Target** | 10-15 minutes | 15-25 minutes |

## Skills You'll Develop

### Programming Fundamentals

- **Iteration**: For loops, while loops, early termination
- **Conditionals**: Modulo operator, divisibility testing
- **Accumulators**: Running sums, products, counts

### Mathematical Thinking

- **Arithmetic series**: Sum formulas, closed-form solutions
- **Number theory basics**: Primes, divisors, GCD/LCM
- **Pattern recognition**: Finding structure in sequences

### Problem-Solving Techniques

- **Bound analysis**: Knowing when to stop searching
- **Precomputation**: Caching values for repeated use
- **Formula derivation**: Converting O(n) to O(1)

## Tips for Foundation Problems

1. **Start simple**: Try the brute force approach first
2. **Look for patterns**: Many problems have mathematical shortcuts
3. **Verify with small inputs**: Test your logic before scaling up
4. **Don't peek at hints**: Struggle builds problem-solving muscle
5. **Time yourself**: Build speed for interviews
